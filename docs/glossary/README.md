This directory holds the canonical vocabulary of Engineering OS — the shared language every other document must use with consistent meaning. It is a language reference, not a behavioral specification: it tells you what a term *means* and how to *use the word*, while object shape lives in the [Domain Model](../architecture/DOMAIN_MODEL.md) and lifecycle behavior lives in the [Company Runtime](../architecture/COMPANY_RUNTIME.md).

The Technical Writer owns this directory and is responsible for keeping terms stable, non-overlapping, and aligned with the authoritative architecture documents. If you find a true conflict between a definition here and those documents, treat it as a defect in the glossary and fix it here.
