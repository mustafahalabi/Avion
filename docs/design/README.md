# Design — Engineering OS

**Status:** Approved
**Version:** 1.0
**Owner:** Product Manager
**Last Updated:** 2026-06-29

---

This directory holds the **design specifications** for Engineering OS — the visual direction and the navigation/layout system that every screen of the product must trace back to. These documents sit between the product's information architecture and its eventual frontend implementation: they prescribe *intent, structure, and behavior*, not framework-specific components, routes, or CSS. Any compliant frontend, in any stack, should be able to satisfy them.

They are owned by the **Product Manager** and are downstream of the [architecture](../architecture/) and [CEO-experience](../ceo-experience/) documents.

## Documents

| Document | Description |
|---|---|
| [VISUAL_FOUNDATION.md](./VISUAL_FOUNDATION.md) | Brand attributes, visual principles, and craft direction — the durable visual intent every interface must trace back to, set before any design system is built. |
| [NAVIGATION_AND_LAYOUT.md](./NAVIGATION_AND_LAYOUT.md) | The navigation and layout system: the app shell, navigable regions, how the CEO moves between them, and the layout archetypes every screen is composed from. |
