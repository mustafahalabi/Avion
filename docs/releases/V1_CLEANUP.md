## Potentially Unused or Orphaned

### `src/app/register/page.tsx`
- A standalone `/register` route exists alongside Clerk's `/sign-up/[[...sign-up]]`
- The register page may be a pre-Clerk stub that is no longer reachable in normal user flows
- **Recommendation**: Verify whether `/register` is linked anywhere; if not, remove in V2

### `src/app/page.tsx` (root landing page)
- The root `/` route exists but all authenticated users are redirected through Clerk's flow
- Verify whether this page serves as a public landing page or is an unreachable stub
- **Recommendation**: Decide if a public marketing/landing page is needed for V2; otherwise redirect to sign-in

### `src/generated/prisma/` (generated files)
- The entire `src/generated/prisma/` directory is auto-generated by Prisma
- It should be in `.gitignore` unless there is a deliberate reason to commit generated files
- **Recommendation**: Confirm `.gitignore` correctly excludes generated files; do not manually edit these

## Duplicate / Overlapping Models

### `Event` vs `TimelineEntry`
- Two separate models exist for recording activity: `Event` and `TimelineEntry`
- `TimelineEntry` has `entityType`, `entityId`, `eventType`, `summary`, `actorId`, `metadata`
- `Event` has `title`, `description`, `type`, `entityType`, `entityId`, `actorId`, `metadata`
- The distinction is unclear; both may serve the same purpose
- **Recommendation**: In V2, consolidate to a single `AuditEvent` or `TimelineEntry` model

### `Memory` vs `Knowledge`
- Both serve as repositories of structured information
- `Memory` is company-scoped with categories; `Knowledge` is global
- Only `Memory` has a UI; `Knowledge` is infrastructure-only in V1
- **Recommendation**: In V2, decide if both are needed or if one model covers all use cases; wire `Knowledge` to AI retrieval
